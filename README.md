# Flappy Bird — COAL Assembly Project

A Flappy Bird clone written in C, with a fully documented x86-64 assembly translation.
This is an academic project for **Computer Organization and Assembly Language (COAL)**.

**Platform:** Windows only (uses GDI+ and Win32 API)

---

## Project Goal

The game is written in C and compiled with MinGW-W64 g++.
The same C source is also compiled to annotated Intel-syntax x86-64 assembly
using `g++ -S -O0 -masm=intel -fno-omit-frame-pointer -fverbose-asm` (no optimizations)
so the assembly maps 1:1 to the C source and is easy to follow.

Every `.asm` file in `assembly/` is heavily commented to explain:
- what each instruction does at the C level
- the stack frame layout and local variable positions
- the Windows x64 calling convention (RCX/RDX/R8/R9 for args, 32-byte shadow space)
- Win32 and GDI+ API calls and their arguments
- physics, collision, and game logic in plain English

## Requirements

- **Windows** (10 or later recommended)
- **MinGW-W64 g++** with `as` (GNU assembler) on your PATH
  - Download: https://winlibs.com or https://www.mingw-w64.org
- GDI+ and WinMM are part of Windows — no extra installs needed

## How to Build and Run

### C version (play the game)

```batch
build_c.bat
run_c_game.bat
```

### Assembly version (build from the .asm files)

```batch
build_asm.bat
run_asm.bat
```

The `.asm` files in `assembly/` are NOT regenerated by `build_asm.bat` — regenerating
would overwrite all the hand-written comments. The script assembles and links the
existing annotated files directly.

## Project Structure

```
COAL/
  src/
    game.c       — physics, collision, particles, pipes, scoring
    render.c     — GDI+ double-buffered sprite rendering
    audio.c      — Win32 PlaySound WAV playback
    main.c       — Win32 window, message loop, WndProc
    game.h
    render.h
    audio.h
  assembly/
    game.asm     — most educational; explains physics & collision math
    render.asm   — GDI+ draw pipeline; explains C++ vtables & name mangling
    audio.asm    — simplest file; good intro to Win32 API calls in asm
    main.asm     — Win32 message dispatch, switch-to-jump-table translation
  assets/
    sprites_desktop/   — PNG sprite images
    audio/             — WAV sound files
  build/               — object files (git-ignored)
  build_c.bat          — compile C source to flappy_bird_c.exe
  build_asm.bat        — assemble .asm files to flappy_bird_asm.exe
  run_c_game.bat       — launch the C executable
  run_asm.bat          — launch the assembly executable
```

## Assembly Files Overview

| File        | Lines  | What it covers                                         |
|-------------|--------|--------------------------------------------------------|
| `audio.asm` | ~323   | Win32 PlaySound; good first file to read               |
| `game.asm`  | ~2258  | Game loop, gravity, AABB collision, particle system    |
| `main.asm`  | ~818   | WinMain, WndProc, message-dispatch jump table          |
| `render.asm`| ~3868  | GDI+ rendering, C++ name mangling, vtables, RTTI       |

Assembly was generated with:

```
g++ -S -O0 -masm=intel -fno-omit-frame-pointer -fverbose-asm src/FILE.c -o assembly/FILE.asm
```

- `-O0` — no optimizations; assembly mirrors the C source line by line
- `-masm=intel` — Intel syntax (`mov rax, 5`) instead of AT&T (`mov $5, %rax`)
- `-fno-omit-frame-pointer` — keeps `push rbp / mov rbp, rsp` so the stack is clear
- `-fverbose-asm` — adds C variable names as comments next to registers

## Game Features

- 1280x720 window with GDI+ double buffering (no flicker)
- 3-frame bird animation (down/mid/up flap)
- Procedural pipe generation with increasing speed
- Particle effects on collision and scoring
- Medal system: Bronze (10+), Silver (20+), Gold (30+), Platinum (40+)
- High score saved to `flappy_highscore.dat`
- Scrolling ground

## Troubleshooting

**Game does not start:**
- Confirm `assets/sprites_desktop/` has all PNG files
- Confirm `assets/audio/` has all WAV files
- Confirm `g++` and `as` are on your system PATH

**Assembly build fails:**
- Run `as --version` to confirm GNU assembler is available
- Run `g++ --version` to confirm MinGW-W64 g++ is installed
